# CodeSearch Issues Log

## 2025-06-18 — Default Energy Weights Externalization session

### Issue 1: module-summary fails for indexed file
- Command: `codesearch module-summary src/core/suggestion-loader.ts`
- Result: "Module not found: src/core/suggestion-loader.ts"
- After `codesearch reindex-files src/core/suggestion-loader.ts` (reported success), same result.
- Note: `search-files "suggestion-loader"` DOES find the file. module-summary still cannot resolve it.
- Workaround: Used `get-file-content` or PowerShell `Get-Content`

### Issue 2: get-file-content encoding error (charmap codec)
- Command: `codesearch get-file-content src/core/suggestion-loader.ts --start-line 1 --end-line 60`
- Result: "Error reading 'src/core/suggestion-loader.ts': 'charmap' codec can't decode byte 0x90 in position 332: character maps to <undefined>"
- Same error for `src/core/genre-profile-types.ts`
- Likely cause: Files contain UTF-8 characters (═, ─, →, ∈) that the Python charmap codec cannot decode. The codesearch CLI probably isn't forcing UTF-8 when reading files.
- Workaround: Used PowerShell `Get-Content -Encoding UTF8`

### Issue 3: get-symbol-code triggers "interactive editor" warning
- Command: `codesearch get-symbol-code suggestion-loader`
- Result: Kiro IDE warns it "appears to be an interactive editor" and blocks execution
- Workaround: Added `ignoreWarning: true` to subsequent codesearch calls
